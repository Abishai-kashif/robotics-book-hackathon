# Citation Management Guide for Physical AI & Humanoid Robotics Textbook

## Overview
This guide provides standards and practices for citing sources in the Physical AI & Humanoid Robotics textbook. Proper citation ensures academic integrity and helps students access original sources for deeper learning.

## Citation Format Standards

### Academic Papers
```
Author, A. A., Author, B. B., & Author, C. C. (Year). Title of the paper. *Journal Name*, Volume(Issue), pages. https://doi.org/xx.xxxx/xxxxx
```

### Technical Documentation
```
Organization. (Year). Document title. *Platform/Software Name*. URL
```

### Books
```
Author, A. A. (Year). *Title of work*. Publisher.
```

### Online Resources
```
Author, A. A. (Year, Month Date). Title of webpage. *Website Name*. URL
```

## Required Citation Elements

### For Academic Papers:
- All author names (or "Author et al." for 4+ authors)
- Publication year
- Complete paper title
- Journal/conference name
- Volume and issue numbers
- Page numbers
- DOI when available

### For Technical Documentation:
- Organization/Company name
- Document/Article title
- Platform/Software name
- Publication date (if available)
- Complete URL

### For Books:
- Author names
- Publication year
- Complete book title
- Publisher information
- Edition (if not first edition)

### For Online Resources:
- Author/organization name
- Publication date (or "n.d." if not available)
- Complete page/article title
- Website/platform name
- Complete URL
- Access date for frequently changing content

## Quality Standards for Sources

### Academic Papers
- Peer-reviewed journals or conferences
- Published within the last 5 years (exceptions for foundational papers)
- From reputable publishers (IEEE, ACM, Springer, Elsevier, etc.)
- High impact factor journals preferred

### Technical Documentation
- Official documentation from technology creators
- Current versions of software/technologies
- Well-maintained and updated resources
- From authoritative sources (ROS.org, NVIDIA developer docs, etc.)

### Books
- Academic or professional publishers
- Recent editions (within 5 years) for technical subjects
- Established authors in the field
- Comprehensive coverage of topics

## Citation Management Process

### 1. Source Evaluation
Before citing any source, verify:
- [ ] Authority and credibility of the source
- [ ] Currency (especially for technical topics)
- [ ] Accuracy and reliability of information
- [ ] Relevance to Physical AI & Humanoid Robotics

### 2. Information Gathering
Collect all required citation elements:
- [ ] Author(s) information
- [ ] Title and subtitle
- [ ] Publication details
- [ ] Access information (URLs, DOIs)
- [ ] Date information

### 3. Format Verification
Ensure citations follow the appropriate format:
- [ ] Complete citation with all required elements
- [ ] Consistent formatting throughout
- [ ] Proper punctuation and capitalization
- [ ] Working URLs and valid DOIs

### 4. Integration into Content
Place citations appropriately:
- [ ] In-text citations where sources are referenced
- [ ] Complete citations in the references section
- [ ] Proper attribution for all borrowed content
- [ ] Distinguishing between primary and secondary sources

## Minimum Source Requirements per Chapter

### Academic Papers
- Minimum 3 peer-reviewed academic papers
- Coverage of foundational and recent developments
- Diverse perspectives on the topic

### Technical Documentation
- Minimum 1 official technical documentation source
- Current version of relevant technologies
- Practical implementation resources

### Books
- Minimum 1 authoritative book (if applicable to topic)
- Comprehensive coverage of subject matter
- Established reference in the field

### Online Resources
- As needed for current developments and tools
- From reputable organizations or experts
- Properly maintained and current resources

## Special Considerations for Physical AI & Humanoid Robotics

### ROS 2 Documentation
- Always cite the official ROS 2 documentation
- Include specific ROS 2 distribution version
- Reference both conceptual and API documentation

### Simulation Platforms
- Cite official documentation for Gazebo, Unity, Isaac Sim
- Include version numbers for simulation software
- Reference specific tutorials or examples used

### Hardware Specifications
- Cite manufacturer documentation for humanoid robots
- Include technical specifications and capabilities
- Reference official hardware documentation

## Example Citations

### Academic Paper Example:
```
Khatib, O., Park, H., Forrai, A., Docquier, G., Bessonnet, G., & Zeghar, M. (2003). Humanoid robots: Making a humanoid face tracking a ball in a juggling task. *IEEE Robotics & Automation Magazine*, 10(4), 36-44. https://doi.org/10.1109/MRA.2003.1256297
```

### Technical Documentation Example:
```
Open Robotics. (2023). ROS 2 Documentation. *ROS 2*. https://docs.ros.org/en/humble/
```

### Book Example:
```
Siciliano, B., & Khatib, O. (Eds.). (2016). *Springer handbook of robotics* (2nd ed.). Springer.
```

### Online Resource Example:
```
NVIDIA. (2023). Isaac ROS: Accelerated Robot Perception and Navigation. *NVIDIA Developer*. https://developer.nvidia.com/isaac-ros-gems
```

## Verification Checklist

Before finalizing any chapter:
- [ ] All sources cited in-text are included in references
- [ ] All references have been properly formatted
- [ ] Minimum source requirements met (5+ sources per chapter)
- [ ] Sources meet quality standards
- [ ] Citations support Physical AI & Humanoid Robotics content
- [ ] No broken URLs or invalid DOIs
